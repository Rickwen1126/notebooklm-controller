# Docs Index

## Purpose

- `docs/` stores project-facing notes that are meant to be readable and reviewable.
- Canonical product and implementation baselines still live in `specs/`.
- `docs/todo/` stores day-scoped working notes, open questions, temporary decisions, and next-step suggestions that are useful for current execution but are not yet canonical.

## Current Categories

- `docs/todo/`: day notes and active working observations
- `docs/reference/`: reusable working references that are not product spec, including curation rules, operational heuristics, and medium-term guidance
